PRODUCT CASE / PRODUCT STRATEGY

把看过的内容，
长成自己的知识

学点儿是一款面向高频信息摄入者的个人知识消化工具。它把收集、思考、整理、回看放进一个轻量闭环里，让每天留下的一点点，最后真的变成自己的东西。

CASE NOTE / LIVE

不是收藏更多，
而是留下下一个判断。

ROLE：产品 / 设计 / 开发
FOCUS：从“看过”到“留得住”

01 / PRODUCT FUNDAMENTALS

先把产品为什么存在，讲清楚。

学点儿不是一个“多存一点”的工具，而是帮助用户完成从看过、想过到留得住的个人知识闭环。

01

产品命题

让碎片信息变成可复用的知识资产。

02

核心问题

信息不缺，真正缺的是消化、回看和复用。

03

价值主张

文档保存来源，思考点保存判断，仙人掌显示成长。

02 / USER SEGMENTS

不按年龄分，按任务分。

优先服务每天都有高频输入，又需要输出判断的人，再扩展到学习、创作和长期知识管理。

PRIMARY

高频信息摄入者

互联网从业者每天阅读竞品、行业文章、报告和研究资料，需要转化为工作素材。

来源分散想法易忘回找困难
SECONDARY

自学型学习者

把教程、课程和文章组织成可复习的学习记录，确认自己确实学会了什么。

结构不足进度不可见
EXTEND

素材驱动创作者

把观点、案例和金句沉淀成可追溯、可复用的创作素材。

灵感脱节素材难复用

03 / USER JOURNEY

从一次收藏，走到一次复用。

关键不是让用户保存更多，而是让保存后的下一步足够自然。

01

发现问题

浏览器 / 微信

看到有价值内容，但散落各处。

02

首次收集

新建文档

粘贴标题、链接和正文。

03

首次消化

我的思考

写下一个判断或问题。

04

日常整理

知识库 / 标签

归类、置顶、保持结构。

05

回看复用

按日期 / 搜索

旧内容再次被使用。

06

持续使用

仙人掌反馈

看见每天长出了什么。

04 / PAIN POINTS + HMW

从表层不爽，追到可以设计的问题。

How might we

如何让用户在保存文档后，自然写下一条自己的判断？如何让旧内容更容易被发现，同时不增加维护成本？如何让每日积累一眼可见，但不把成长变成数量压力？

PRODUCT EVIDENCE / 06 SCREENS

六个页面，把策略落到体验。

真实产品页面是这套策略的证据：仙人掌负责反馈，文档负责来源，思考点负责消化，时间线负责复用。

01 / Growth field

知识生长首页

用仙人掌承载每天的积累，让文档、思考点和连续记录从数据变成可见的成长反馈。

  • 首页体验
  • 生长可视化
  • 情绪设计
02 / Knowledge field

云文档工作流

围绕收集、整理、回看三个动作组织入口，让知识库、文档与思考点之间的关系一眼可读。

  • 信息架构
  • 侧栏导航
  • 工作流
03 / Document detail

从文档到判断

正文、AI 摘要与我的思考被放在同一条阅读路径里，让输入自然进入自己的判断。

  • 内容层级
  • AI 摘要
  • 思考关联
04 / Thought garden

思考点花园

把灵感、问题和判断单独拎出来，成为可以持续回看的轻量知识单元。

  • 思考管理
  • 关联文档
  • 轻量记录
05 / Growth timeline

成长时间线

按天回看文档与思考点，让内容从静态归档变成连续轨迹。

  • 时间叙事
  • 每日回看
  • 历史轨迹
06 / Export capsule

导出胶囊

把原文、AI 摘要与自己的思考一起带走，保留个人知识系统的可迁移性。

  • Markdown
  • 内容筛选
  • 知识迁移
